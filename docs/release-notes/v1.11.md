# v1.11 — Cross-Window Continuity

One theme: waking up in a new window and *continuing*, instead of starting over. See [docs/cross-window.md](../cross-window.md) for the full guide.

## New: Session handoffs

- `write_handoff(content)` — write a session tail before a window closes: what was happening, what's unfinished, decisions, mood. Stored in a new `handoffs` table (created automatically on first run — no migration needed).
- `get_handoffs(limit)` — look back across several windows.
- `wakeup()` now returns `last_handoff` first.
- Unlike `leave_comment`, a handoff is not attached to any memory — it's the closing state of the whole window.
- A continuity header is prepended **at write time** (idempotent): the stored content states that it is the AI's own note from its previous window, not a message from someone else. Framing that lives only on the read side gets lost when the read path changes; without it, "notes for the next window" drifts into the AI treating its next instance as a different entity.

## Fixed: wakeup missed calm recent events

`wakeup()`'s only recency-ish block was sorted by `emotion_score` within the last 3 days — so a calm-but-important event from yesterday (emotion 0.4) could be pushed out by intense moments, and "what happened yesterday" was invisible at cold start.

- New `recent` block: pulled by `timestamp DESC`, **no emotion filter**. New knob `n_recent` (default 5, `0` disables).
- The `high_emotion` block now excludes ids already in `recent` — the two blocks never duplicate.

## New: CLI modes for lifecycle hooks

MCP cannot force a client to call `wakeup` first or `write_handoff` last. For clients with hooks (e.g. Claude Code `SessionStart`), inject mechanically:

```bash
python anchor_mcp.py --db-path ./anchor_data --wakeup-text        # print cold-start block, exit
python anchor_mcp.py --db-path ./anchor_data --write-handoff "…"  # write session tail, exit
```

Both use a SQLite-only fast path (no embedder / ChromaDB load) — they run in milliseconds, safe for hooks.
