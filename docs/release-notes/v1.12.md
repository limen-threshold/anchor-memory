# v1.12 — Cross-Window Continuity, machine-side

One theme: waking up in a new window and *continuing*. The architecture mirrors a production companion-AI gateway that has run this pattern for months. Full guide: [docs/cross-window.md](../cross-window.md).

(v1.11 was published and retracted the same day — it put continuity on the model's discipline: a tool the AI had to remember to call before the window closed. Windows don't close gracefully. v1.12 is the machine-side rebuild.)

## New: `anchor_proxy.py` — OpenAI-compatible continuity proxy

Sits between your frontend (Open WebUI / SillyTavern / LobeHub / anything OpenAI-compatible) and any OpenAI-compatible upstream. Every turn, machine-side:

- **Pinned layer → system**: identity files + `session_state.md` + `recent_timeline.md`, in `_order.txt` order (stable → cacheable)
- **Previous-window tail**: `last_session.md` injected on fresh windows; **overwritten after every response**, so it survives crashes, closed tabs, dead phones
- **Time + interval block** into the last user message (not system — a per-turn system block breaks upstream prompt caching)
- **Per-turn recall**: every user message searches memory; results injected as `<relevant_memories>` with dates + a temporal anchor ("that date is when it happened, not now")
- **Curator** (BYO-LLM): background judge-store after each response, multi-event per exchange, appends `recent_timeline.md` event lines
- **Intent-split gate** (BYO-LLM): long messages split into separate search intents before recall

**Zero-LLM floor**: everything except gate/curator runs with no LLM beyond your chat model. Gate/curator light up via `ANCHOR_LLM` / `~/.anchor/config.yaml` (anchor_llm.py, same BYO pattern as dream pass) — the AI can ask its human which provider to use and write the config itself.

Server deps (`pip install fastapi uvicorn httpx`) are needed only for the proxy.

## New: `anchor_pinned.py` — the file contract

session_state (write-side continuity header, auto-archive rotation), recent_timeline (`[YYYY-MM-DD HH:MM] event`), last_session tail (`**User**:`/`**Assistant**:`), `_order.txt` manifest. Plain markdown, stdlib-only module — **this is the modding seam**: anything that can read/write a text file can integrate (browser extensions, shell hooks, other gateways). Pipeline functions in anchor_proxy.py are individually replaceable; `build_turn()` is importable for embedding in your own server.

## MCP additions (works everywhere, including web clients)

- `write_session_state` — the AI's own rolling state, auto-archived, continuity-headered. The model-side layer works over plain MCP on any platform.
- `wakeup()` now also returns `session_state`, `recent_timeline`, and `last_session` when the pinned files exist, plus a new `recent` block (below).
- `store_memory` gains `context` — two-layer storage (text = embedded summary, context = verbatim source). The DB column existed for a long time; `store()` never exposed it.
- `--pinned-dir` flag (default `<db-path>/pinned`); `--wakeup-text` CLI mode for session-start hooks (SQLite-only fast path, milliseconds).

## Fixed: wakeup missed calm recent events

The only recency-ish block in `wakeup()` was emotion-sorted within 3 days — a calm-but-important event from yesterday could be pushed out by intense moments, making "what happened yesterday" invisible at cold start. New `recent` block: `timestamp DESC`, **no emotion filter**, knob `n_recent` (default 5); `high_emotion` dedups against it.
