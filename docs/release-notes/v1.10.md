# Anchor Memory v1.10

两个 bug 修复 + 两个新特性。bug 来自下游 Limen/Ren 记忆栈复现时实测坐实（Anchor 是它们的上游，所以 bug 也镜像了过去）；特性是同一批 v5 工作的回灌。

## 为什么修 —— bug 1：重存一条记忆会啃光它的边

`insert()` 用 `INSERT OR REPLACE`。SQLite 的 REPLACE 在主键冲突时是**先 DELETE 旧行再 INSERT**，而 DELETE 会触发 `edges` 表的 `ON DELETE CASCADE` —— 把这条记忆的**所有边**（Hebbian 历史 + 手动连边）全部级联删掉。

也就是说，每次 re-store 一条已存在的记忆（改个 text、换个 tag、修个 tier），它在图里的全部连接当场蒸发。

实测坐实：

```
connect(A, B, 2.0)        # edge A↔B = 2.0
insert(A, "A edited")     # 重存 A
get_edge_weight(A, B)     # → None   ← 边没了
```

## 为什么修 —— bug 2：记忆能连到自己

`_upsert_edge()` 不挡 `source == target`。于是 `connect(A, A)` 会写出一条 A→A 自环边，`get_neighbors(A)` 把 A 自己当邻居返回 —— associative recall 那一跳会把一条记忆**自己**当「相关记忆」再喂一遍，自我强化的污染。

## 怎么修的

**bug 1**：`insert()` 从 `INSERT OR REPLACE` 改成原生 UPSERT（`ON CONFLICT(memory_id) DO UPDATE`）。冲突时只 UPDATE 字段、不 DELETE 行 → 不触发级联 → 边活着。两个有独立生命的字段用 `COALESCE` 保留（只有首写设）：

- `timestamp` —— 一次修订不该让老记忆显得「新」（否则破坏按时间排序 / recency boost）；
- `emotion_score` —— 情绪传播 / dream pass own 它，改个 text 不该重置情绪。

**bug 2**：`_upsert_edge()` 开头加一道结构性护栏 `if source_id == target_id: return`。所有写边入口（`connect` / `connect_batch` / 新增的 `migrate_edges`）都汇流过它，一道护栏堵死所有路径。

## 新特性 —— recency boost（第三项排序加成）

`search()` 在 citation / emotion 之外加第三项近因加成：同样相关的记忆，**近的更容易浮上来**。指数半衰：

```
recency_boost = recency_weight · 0.5 ^ (age_days / recency_halflife_days)
```

两个旋钮，是 `AnchorMemory` 的实例属性，可随时调：

- `recency_weight`（默认 `0.05`）—— age 0 时的最大加成，约 citation 上限（0.15）的 1/3。设 `0` 即关闭。
- `recency_halflife_days`（默认 `30`）—— 加成每 N 天减半。

跟 citation / emotion 一样从 `score` 减去（distance 语义，越小越靠前）。`debug=True` 的输出里多一项 `recency_boost`。

## 新特性 —— dedup merge（去重合并）

两条记忆指向同一件事（典型：两个子系统各存了一遍同一事件）时，把它们合二为一：

```python
mem.merge_memories(survivor_id, duplicate_id)
```

把 `duplicate` 折叠进 `survivor`、然后从两个存储（SQLite 行 + ChromaDB 向量）都删掉 duplicate。survivor 保留自己的 text / 向量 / id，只合并元数据。**调用方决定谁活**（通常选更早那条，但质量可以 override）。

合并规则：

- **edges**：survivor 继承 duplicate 的边（`migrate_edges`）；撞键饱和加（`min(和, MAX_EDGE_WEIGHT)`），自环被上面的护栏丢弃。否则 duplicate 的 Hebbian 历史会随删除蒸发。
- **usage_count**：相加。
- **timestamp**：取更早（记忆和它最老的见证一样老）。
- **pinned**：取或。
- **emotion_score**：取较高。
- **tag / tier / text / context**：保留 survivor 自己的（是折叠元数据，不是重写本体）。

合并写一条 `merged` event 进 event log（survivor 为主、detail 里记 `from=<duplicate>` 及各项合并结果）。

## Breaking changes

无。

- `insert()` 首写行为完全不变；只是重存时不再删边（纯改进）。
- 自环护栏只拒绝本就不该存在的自环边，正常边路径零变化。
- `recency_weight` 默认非零，会**轻微**改变排名（近的略靠前）；想保持旧排序设 `mem.recency_weight = 0`。
- `merge_memories` / `migrate_edges` 是纯新增 API。

## 验证

全部在临时库上贴返回逐行核（行内 probe，非单测框架）：

```
# bug 1 — 重存不再啃边
connect(A,B,2.0) → insert(A,"edited") → edge A↔B = 2.0 ✓
  内容更新 ✓  timestamp 保留 ✓  emotion 保留 ✓

# bug 2 — 自环
connect(C,C,1.5) → edge C-C = None ✓   connect(A,C) 正常边 = 存活 ✓

# recency — 开关对照证明真参与打分
两条同义记忆，一条 age~0、一条 age~60d：
  recency ON : new boost=0.0500 score=-0.1000 / old boost=0.0125 score=-0.0220
  recency OFF: new boost=0.0000 score=-0.0700 / old boost=0.0000 score=-0.0295
  → 开关切换 score 明显变化，排名 new 在 old 前

# merge
survivor S(usage2,pin0,emo0.3,连X w1.0) + duplicate D(usage5,pin1,emo0.9,连Y w2.0、连X w1.5、连S w3.0)
  → edges_migrated=6, usage=7, pinned=1, emotion=0.9, ts=S原值（取早）
  → S-X=2.5（1.0+1.5 撞键饱和加） S-Y=2.0（迁来） S-S=None（自环挡掉）
  → duplicate 双表删除 ✓   merged event 已记 ✓
```
